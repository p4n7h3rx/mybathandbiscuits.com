# Hostinger Node.js Hosting Guide

This guide ensures your Bath & Biscuits website runs smoothly on Hostinger's Node.js hosting platform.

## 1. Project Structure
To run correctly on Hostinger, your project should have these key files in the root:
- `dist/` - The compiled frontend (generated by `npm run build`)
- `server.js` - The production Express server
- `package.json` - Includes dependencies and start scripts

## 2. Configuration Settings (Hostinger Dashboard)
In your Hostinger Node.js application settings, use these values:
- **Node.js version**: 18 or higher (20+ recommended)
- **Application Mode**: Production
- **Application Startup File**: `server.js` (or leave blank if it uses `package.json`)
- **Output Directory**: Leave as `null` or clear it if it's a Node app.

## 3. Deployment Steps

### Method A: Automated Build (Recommended)
1. Upload your code to Hostinger (via Git or File Manager).
2. Run `npm install`.
3. Run `npm run build`.
   - *Note: If you see "Permission Denied", I've added a fix in `package.json` that handles this automatically.*

### Method B: Local Build (Most Reliable)
If the server build fails, do this locally:
1. Run `npm run build` on your computer.
2. Upload the `dist` folder to Hostinger.
3. Ensure `app.js` and `package.json` are also uploaded.

## 4. Common Troubleshooting

### 503 Service Unavailable
- **Cause**: The Node app crashed or failed to start.
- **Fix**: Check that `server.js` is set as the startup file and that `npm install` was successful.

### Missing Assets (Favicon, Images)
- **Cause**: Incorrect path resolution in the server.
- **Fix**: The provided `app.js` uses absolute paths (`path.join(__dirname, 'dist')`) to ensure assets are found regardless of where the script is executed.

### Permission Denied (Vite)
- **Cause**: Linux systems often restrict execution permissions in `node_modules`.
- **Fix**: Use the updated build command: `chmod +x node_modules/.bin/vite && vite build`.
